嗨，朋友们，

这周 GitHub 发了两个更新，主题一致：让你每天在做的事情更快。不用学新功能，就是少等。

性能优化这种事真的太不受重视了。没人会在 Twitter 上发长帖吹嘘自己砍掉了几毫秒的加载时间。但当你的 issue 秒开、PR review 不卡了？这种改进是每天都在累积的。

## 🚢 新动态

### GitHub Issues 加载更快了

GitHub Issues 快了一大截。35% 的 issue 页面加载时间低于 200ms，年初这个数字才 2%。这不是渐进式优化，这是体验质变。

现在 [github.com](https://github.com/) 上已经生效了。登录就行，不需要手动开启。在你的仓库 issue 列表、Issues 面板、GitHub Projects 里都能感受到。团队说这只是一系列性能优化的第一步，后面还有更多。

### PR「Files Changed」新体验正式上线

还记得我在 [十一月](https://mainbranch.dev/newsletter/issue-1/) 介绍过的新版 Files Changed 体验吗？现在已经成为所有人的默认设置了，不用再手动开启。

亮点：你可以在任意行评论（不只是改动行），文件树可以拖拽调整大小还有评论和错误标记，草稿评论会本地保存所以浏览器崩了也不丢。还有一个针对大 PR 的实验性虚拟化模式，让页面保持流畅。不喜欢的话还是可以切回去，但建议先用一周再说。

![大 PR？新的虚拟化视图让页面保持流畅。](/images/pr-files-changed.png)

## 📺 在看什么

### Open Source Friday: Clawdbot 创始人访谈

有点"自卖自夸"的意思，不过我跟 [Clawdbot](https://clawdbot.com/) 的创始人聊了一次特别好的对话。[Peter Stenberger](https://x.com/steipete) 是个很酷的人，退休之后又出山做了这个工具，对 AI 保持着一种基础优先的态度，我真的很欣赏。没有炒作，就是实用，安全性也拉满。

如果你想听一个老江湖聊怎么做 AI 工具而不跑偏，值得一看。

## 🔧 在用什么

[Copilot SDK](https://github.blog/changelog/2026-01-14-copilot-sdk-in-technical-preview/) 刚出了技术预览，我玩得不亦乐乎。我做了一个"issue 版 Tinder"原型。[没错，真的。](https://x.com/acolombiadev/status/2014830414410518885)

下一期我会深入聊 Copilot CLI。它能帮你搞定那些枯燥的活儿，让你专注于有意思的部分。

## ✨ 这周

上周末我进入了极度专注模式，一口气完成了一个西班牙语大师课（后面再细说）。效率是很高，但对身边人不太友好（老公和孩子们，抱歉了）。一进入这种状态我就忘了吃饭、睡觉、运动、做人。同为 ADHD 脑的朋友们应该懂。这周咱们找点平衡吧，互相监督。

就这些。性能优化不炫酷，但它每天都在累积。

如果觉得有用，转给你的团队吧。回复告诉我你想看什么内容。

感恩，下周见，
